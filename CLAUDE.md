# CLAUDE.md｜Other-Note 筆記庫

> 個人技術筆記知識庫（繁體中文）。本檔只放行為約束；索引見 `00_索引.md`，格式規範見 `00_sample/00_筆記規範.md`。

## 目錄結構

- `00_sample/`：範本與筆記規範
- `00_概念/`：概念、方法論（數學、硬體…）
- `00_其他(工具_應用程式_網站)/`：工具、應用程式、網站、LLM
- `01_程式語言/`：各語言筆記
- `02_作業系統/`、`03_伺服器服務/`、`04_雲端平台(CloudPlatform)/`、`05_樹莓派/`

## 查找筆記（先索引、後搜尋，控 token）

1. 先讀 `00_索引.md` 直接定位目標檔案。
2. 索引沒有 → `find . -name "*關鍵字*"` 或 `grep -rn`，**禁止**無定位掃整個目錄。
3. 只讀本次任務相關的筆記，不順手瀏覽。

## 更新與新增

- **先更新、後新增**：已有主題相符的筆記 → 補章節，不建新檔。
- **內容歸位**：補內容須放語意相符的章節（安裝指令→安裝、設定→配置文檔、用法→指令、錯誤排除→例外狀況），詳見規範「更新既有筆記」。
- 無適合筆記 → 依 `00_sample/00_筆記規範.md` 建立（範本對照、命名、結構、總覽規則都在該檔）。
- **單一真相**：內容過時或錯誤 → 直接刪修原檔，不留舊段落、不建 `_v2` / `_copy` / `_backup` 副本；歷史交給 Git。
- **跨分類**：筆記同時符合多種分類 → 本體只放最符合的目錄，其他相關筆記在 `## 參考資料` 下建相對路徑連結指向它，不複製內容；移動/改名時同步修正引用連結。
- 新增、移動、改名筆記後 → **必須同步更新 `00_索引.md`**；新的常查主題也補進索引。
- 同目錄出現 3 個以上同類工具筆記 → 依規範建立 `_總覽.md` 並登記到索引。
- 檔名避免含 `test`（`.gitignore` 排除 `*test*.*`，會不入版控）。

## 邊界

- 不讀、不動：`.git/`、`node_modules/`、`.DS_Store`、`note_temp/`（暫存區，非筆記）、`note_tool.*`。
- **Submodule 唯讀**：`01_程式語言/` 下各 `*-Code` / `*_Book_Code`、`00_其他.../LLM(大型語言模型)/0_制度文件(規章制度)/` 皆為外部 repo，只讀不改，搜尋時排除。
- 筆記內不得出現金鑰、token、密碼明文（引用一律 `$ENV_VAR` 形式）。
- `00_sample/` 是範本目錄：修範本前先確認影響範圍，不放測試垃圾檔。

## 語言與風格

- 章節標題繁體中文；程式碼、指令、識別字保留英文。
- 說明文字精簡條列，可執行的內容一律用 code block。

## 相關制度

- AI 協作制度（S/W/R 派工、權限邊界、token 紀律）在 `ai-governance` repo，本 repo 以 submodule 掛載於 `00_其他(工具_應用程式_網站)/LLM(大型語言模型)/0_制度文件(規章制度)/`；本檔不重複其內容，跨 repo 任務遵守該套 A~M。
