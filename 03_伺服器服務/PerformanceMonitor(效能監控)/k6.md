# 核心概念

| 名詞 | 說明 |
|---|---|
| **VU（Virtual User）** | 模擬並發使用者數量 |
| **Iteration** | 每個 VU 執行一次 `default function` |
| **Duration** | 測試持續時間 |
| **Ramp-up / Ramp-down** | VU 數量逐漸增加 / 減少的階段 |
| **Threshold** | 測試通過 / 失敗的指標條件（如 p95 < 500ms） |
| **Check** | 斷言回應內容（類似 assert） |
| **Metric** | k6 自動收集的量測數據（http_req_duration、error_rate 等） |

**常用內建 Metrics：**

| Metric | 說明 |
|---|---|
| `http_req_duration` | HTTP 請求總時間（含等待） |
| `http_req_failed` | 失敗請求比例 |
| `http_reqs` | 總請求數（RPS） |
| `vus` | 目前活躍 VU 數 |
| `iterations` | 完成 iteration 總數 |

## 配置文檔

### 基本測試腳本結構

通常命名為 `script.js`（或任意 `.js` 檔名）

```javascript
import http from 'k6/http';
import { sleep } from 'k6';

// options：測試配置（可移至 CLI 參數）
export const options = {
  vus: 10,           // 並發 VU 數
  duration: '30s',   // 測試持續時間
};

// 每個 VU 每次 iteration 執行此函式
export default function () {
  http.get('https://api.example.com/users');
  sleep(1);  // 模擬使用者思考時間（秒）
}
```

### Options 設定

```javascript
export const options = {
  // 固定 VU 數
  vus: 50,
  duration: '1m',

  // 或使用 stages（階段式）
  stages: [
    { duration: '30s', target: 20 },   // 爬升到 20 VU
    { duration: '1m',  target: 20 },   // 維持 20 VU
    { duration: '10s', target: 0 },    // 降回 0
  ],

  // Threshold：測試通過條件
  thresholds: {
    http_req_duration: ['p(95)<500'],  // 95% 請求 < 500ms
    http_req_failed: ['rate<0.01'],    // 失敗率 < 1%
  },
};
```

# 指令

## 執行測試

```bash
# 基本執行
k6 run script.js

# 覆寫 options（CLI 優先於腳本內 options）
k6 run --vus 20 --duration 1m script.js

# 使用 .env 環境變數傳入腳本
k6 run -e API_URL=https://api.example.com script.js

# 靜默模式（不顯示即時 metrics）
k6 run --quiet script.js

# 列出可用 API
k6 version
```

## 輸出格式

```bash
# 輸出 JSON（完整結果）
k6 run --out json=result.json script.js

# 輸出到 InfluxDB（搭配 Grafana）
k6 run --out influxdb=http://localhost:8086/k6 script.js

# 輸出到 Prometheus Remote Write
k6 run --out experimental-prometheus-rw script.js

# 輸出到 CSV
k6 run --out csv=result.csv script.js
```

# 測試腳本範例

## HTTP GET / POST

```javascript
import http from 'k6/http';
import { check, sleep } from 'k6';

export default function () {
  // GET 請求
  const res = http.get('https://api.example.com/users/1');

  check(res, {
    'status is 200': (r) => r.status === 200,
    'response time < 500ms': (r) => r.timings.duration < 500,
  });

  sleep(1);

  // POST 請求（JSON body）
  const payload = JSON.stringify({ name: 'test', email: 'test@example.com' });
  const params = { headers: { 'Content-Type': 'application/json' } };

  const postRes = http.post('https://api.example.com/users', payload, params);

  check(postRes, {
    'created successfully': (r) => r.status === 201,
  });
}
```

## Threshold（通過條件）

```javascript
export const options = {
  thresholds: {
    // 95% 請求回應時間 < 500ms，否則測試失敗
    'http_req_duration': ['p(95)<500', 'p(99)<1000'],

    // 錯誤率低於 1%
    'http_req_failed': ['rate<0.01'],

    // 特定 URL 的條件（使用 tags）
    'http_req_duration{url:https://api.example.com/users}': ['p(95)<300'],
  },
};
```

## Check（斷言）

```javascript
import { check } from 'k6';

// check 回傳 boolean，不會中斷測試（失敗只記錄）
const success = check(res, {
  'status 200':       (r) => r.status === 200,
  'has body':         (r) => r.body.length > 0,
  'content-type JSON': (r) => r.headers['Content-Type'].includes('application/json'),
  'user id exists':   (r) => JSON.parse(r.body).id !== undefined,
});
```

## Stages（階段式負載）

```javascript
export const options = {
  stages: [
    { duration: '1m',  target: 50  },  // 1 分鐘爬升到 50 VU
    { duration: '3m',  target: 50  },  // 維持 50 VU 3 分鐘（穩定負載）
    { duration: '1m',  target: 100 },  // 衝刺到 100 VU
    { duration: '2m',  target: 100 },  // 維持峰值
    { duration: '1m',  target: 0   },  // 降回 0（冷卻）
  ],
  thresholds: {
    http_req_duration: ['p(95)<800'],
    http_req_failed: ['rate<0.05'],
  },
};
```

## WebSocket

```javascript
import ws from 'k6/ws';
import { check } from 'k6';

export default function () {
  const url = 'ws://localhost:8080/ws';

  const res = ws.connect(url, {}, function (socket) {
    socket.on('open', () => {
      socket.send(JSON.stringify({ type: 'ping' }));
    });

    socket.on('message', (data) => {
      check(JSON.parse(data), {
        'pong received': (d) => d.type === 'pong',
      });
      socket.close();
    });

    socket.setTimeout(() => socket.close(), 5000);
  });

  check(res, { 'connected': (r) => r && r.status === 101 });
}
```

# 應用範例：Python Flask ERP/WMS 壓測

> 適用於 Flask + Gunicorn（2 workers × 2 threads）+ MongoDB + Redis（flask-limiter）架構。
> 壓測目的：找出 Gunicorn 排隊瓶頸、MongoDB 慢查詢、Rate Limit 設定是否合理，作為 k8s HPA replica 數量的決策依據。

## 壓測場景規劃

| 腳本 | 目的 | 關鍵指標 |
|---|---|---|
| `test-auth.js` | 登入並發，模擬上班尖峰 | `p(95) < 300ms`、錯誤率 `< 1%` |
| `test-inventory.js` | 倉庫查詢 / 出入庫混合讀寫 | `p(99) < 1s`、MongoDB 連線池 |
| `test-rate-limit.js` | 驗證 flask-limiter（Redis）是否生效 | 同 IP 高頻後出現 `429` |

## 登入流量壓測

```javascript
// test-auth.js
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '30s', target: 50 },  // 30 秒內爬升到 50 VU
    { duration: '1m',  target: 50 },  // 維持 1 分鐘
    { duration: '10s', target: 0  },  // 收斂
  ],
  thresholds: {
    http_req_failed:   ['rate<0.01'],   // 錯誤率 < 1%
    http_req_duration: ['p(95)<300'],   // 95% 請求 < 300ms
  },
};

const BASE_URL = __ENV.BASE_URL || 'http://localhost:5000';

export default function () {
  const res = http.post(
    `${BASE_URL}/api/auth/login`,
    JSON.stringify({ username: 'admin', password: 'admin123' }),
    { headers: { 'Content-Type': 'application/json' } }
  );

  check(res, {
    'login success':  (r) => r.status === 200,
    'has token':      (r) => r.json('data.access_token') !== undefined,
  });

  sleep(1);
}
```

## 倉庫操作混合壓測

```javascript
// test-inventory.js
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  vus: 100,
  duration: '2m',
  thresholds: {
    http_req_duration: ['p(99)<1000'],  // 99% < 1s
    http_req_failed:   ['rate<0.02'],
  },
};

const BASE_URL = __ENV.BASE_URL || 'http://localhost:5000';

// setup()：在測試開始前執行一次，取得 JWT token
export function setup() {
  const res = http.post(
    `${BASE_URL}/api/auth/login`,
    JSON.stringify({ username: 'admin', password: 'admin123' }),
    { headers: { 'Content-Type': 'application/json' } }
  );
  return { token: res.json('data.access_token') };
}

export default function (data) {
  const headers = {
    Authorization: `Bearer ${data.token}`,
    'Content-Type': 'application/json',
  };

  // 混合讀寫場景：模擬倉庫日常操作比例
  const rand = Math.random();
  let res;

  if (rand < 0.5) {
    res = http.get(`${BASE_URL}/api/inventory/list`, { headers });
  } else if (rand < 0.75) {
    res = http.get(`${BASE_URL}/api/product/list`, { headers });
  } else {
    res = http.get(`${BASE_URL}/api/warehouse/list`, { headers });
  }

  check(res, { 'status 200': (r) => r.status === 200 });
  sleep(0.5);
}
```

## Rate Limit 驗證

```javascript
// test-rate-limit.js
// 驗證 flask-limiter（Redis 後端）是否在高頻請求後回傳 429
import http from 'k6/http';
import { check } from 'k6';

export const options = {
  vus: 1,
  iterations: 30,   // 同一 VU 連打 30 次，預期觸發 Rate Limit
};

const BASE_URL = __ENV.BASE_URL || 'http://localhost:5000';

export default function () {
  const res = http.post(
    `${BASE_URL}/api/auth/login`,
    JSON.stringify({ username: 'wrong', password: 'wrong' }),
    { headers: { 'Content-Type': 'application/json' } }
  );

  // 前幾次 401（帳密錯），超過限制後應出現 429
  check(res, {
    'blocked or auth error': (r) => r.status === 401 || r.status === 429,
  });
}
```

## 執行與分析

```bash
# 執行登入壓測（指定目標 URL）
k6 run -e BASE_URL=http://your-server test-auth.js

# 執行倉庫混合壓測
k6 run -e BASE_URL=http://your-server test-inventory.js

# 驗證 Rate Limit
k6 run -e BASE_URL=http://your-server test-rate-limit.js

# 輸出 JSON 結果供後續分析
k6 run -e BASE_URL=http://your-server --out json=result.json test-inventory.js

# 搭配 Grafana 即時監控（需先啟動 InfluxDB）
k6 run -e BASE_URL=http://your-server \
  --out influxdb=http://localhost:8086/k6 \
  test-inventory.js
```

```
壓測結果判讀：

1. p(95) 超過 500ms → Gunicorn workers 不足，增加 GUNICORN_WORKERS
   或 MongoDB 慢查詢，用 db.setProfilingLevel(1) 找出慢查詢索引缺失

2. 錯誤率 > 1% 且狀態碼 502 → Gunicorn 排隊滿，增加 k8s HPA replicas

3. test-rate-limit 全程無 429 → flask-limiter 未生效，
   確認 REDIS_URL 設定正確、Redis Pod 正常運行

4. p(95) < 200ms 且錯誤率 0% → 可減少 HPA minReplicas，降低 VPS 成本
```
