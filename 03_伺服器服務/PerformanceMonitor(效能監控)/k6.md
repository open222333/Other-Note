# 核心概念

| 名詞 | 說明 |
|---|---|
| **VU（Virtual User）** | 模擬並發使用者數量 |
| **Iteration** | 每個 VU 執行一次 `default function` |
| **Duration** | 測試持續時間 |
| **Ramp-up / Ramp-down** | VU 數量逐漸增加 / 減少的階段 |
| **Threshold** | 測試通過 / 失敗的指標條件（如 p95 < 500ms） |
| **Check** | 斷言回應內容（類似 assert） |
| **Metric** | k6 自動收集的量測數據（http_req_duration、error_rate 等） |

**常用內建 Metrics：**

| Metric | 說明 |
|---|---|
| `http_req_duration` | HTTP 請求總時間（含等待） |
| `http_req_failed` | 失敗請求比例 |
| `http_reqs` | 總請求數（RPS） |
| `vus` | 目前活躍 VU 數 |
| `iterations` | 完成 iteration 總數 |

# 安裝

## Debian (Ubuntu)

```bash
sudo gpg -k
sudo gpg --no-default-keyring \
  --keyring /usr/share/keyrings/k6-archive-keyring.gpg \
  --keyserver hkp://keyserver.ubuntu.com:80 \
  --recv-keys C5AD17C747E3415A3642D57D77C6C491D6AC1D69

echo "deb [signed-by=/usr/share/keyrings/k6-archive-keyring.gpg] \
  https://dl.k6.io/deb stable main" | \
  sudo tee /etc/apt/sources.list.d/k6.list
```

## 配置文檔

### 基本測試腳本結構

通常命名為 `script.js`（或任意 `.js` 檔名）

```javascript
import http from 'k6/http';
import { sleep } from 'k6';

// options：測試配置（可移至 CLI 參數）
export const options = {
  vus: 10,           // 並發 VU 數
  duration: '30s',   // 測試持續時間
};

// 每個 VU 每次 iteration 執行此函式
export default function () {
  http.get('https://api.example.com/users');
  sleep(1);  // 模擬使用者思考時間（秒）
}
```

### Options 設定

```javascript
export const options = {
  // 固定 VU 數
  vus: 50,
  duration: '1m',

  // 或使用 stages（階段式）
  stages: [
    { duration: '30s', target: 20 },   // 爬升到 20 VU
    { duration: '1m',  target: 20 },   // 維持 20 VU
    { duration: '10s', target: 0 },    // 降回 0
  ],

  // Threshold：測試通過條件
  thresholds: {
    http_req_duration: ['p(95)<500'],  // 95% 請求 < 500ms
    http_req_failed: ['rate<0.01'],    // 失敗率 < 1%
  },
};
```

# 指令

## 執行測試

```bash
# 基本執行
k6 run script.js

# 覆寫 options（CLI 優先於腳本內 options）
k6 run --vus 20 --duration 1m script.js

# 使用 .env 環境變數傳入腳本
k6 run -e API_URL=https://api.example.com script.js

# 靜默模式（不顯示即時 metrics）
k6 run --quiet script.js

# 列出可用 API
k6 version
```

## 輸出格式

```bash
# 輸出 JSON（完整結果）
k6 run --out json=result.json script.js

# 輸出到 InfluxDB（搭配 Grafana）
k6 run --out influxdb=http://localhost:8086/k6 script.js

# 輸出到 Prometheus Remote Write
k6 run --out experimental-prometheus-rw script.js

# 輸出到 CSV
k6 run --out csv=result.csv script.js
```

# 測試腳本範例

## HTTP GET / POST

```javascript
import http from 'k6/http';
import { check, sleep } from 'k6';

export default function () {
  // GET 請求
  const res = http.get('https://api.example.com/users/1');

  check(res, {
    'status is 200': (r) => r.status === 200,
    'response time < 500ms': (r) => r.timings.duration < 500,
  });

  sleep(1);

  // POST 請求（JSON body）
  const payload = JSON.stringify({ name: 'test', email: 'test@example.com' });
  const params = { headers: { 'Content-Type': 'application/json' } };

  const postRes = http.post('https://api.example.com/users', payload, params);

  check(postRes, {
    'created successfully': (r) => r.status === 201,
  });
}
```

## Threshold（通過條件）

```javascript
export const options = {
  thresholds: {
    // 95% 請求回應時間 < 500ms，否則測試失敗
    'http_req_duration': ['p(95)<500', 'p(99)<1000'],

    // 錯誤率低於 1%
    'http_req_failed': ['rate<0.01'],

    // 特定 URL 的條件（使用 tags）
    'http_req_duration{url:https://api.example.com/users}': ['p(95)<300'],
  },
};
```

## Check（斷言）

```javascript
import { check } from 'k6';

// check 回傳 boolean，不會中斷測試（失敗只記錄）
const success = check(res, {
  'status 200':       (r) => r.status === 200,
  'has body':         (r) => r.body.length > 0,
  'content-type JSON': (r) => r.headers['Content-Type'].includes('application/json'),
  'user id exists':   (r) => JSON.parse(r.body).id !== undefined,
});
```

## Stages（階段式負載）

```javascript
export const options = {
  stages: [
    { duration: '1m',  target: 50  },  // 1 分鐘爬升到 50 VU
    { duration: '3m',  target: 50  },  // 維持 50 VU 3 分鐘（穩定負載）
    { duration: '1m',  target: 100 },  // 衝刺到 100 VU
    { duration: '2m',  target: 100 },  // 維持峰值
    { duration: '1m',  target: 0   },  // 降回 0（冷卻）
  ],
  thresholds: {
    http_req_duration: ['p(95)<800'],
    http_req_failed: ['rate<0.05'],
  },
};
```

## WebSocket

```javascript
import ws from 'k6/ws';
import { check } from 'k6';

export default function () {
  const url = 'ws://localhost:8080/ws';

  const res = ws.connect(url, {}, function (socket) {
    socket.on('open', () => {
      socket.send(JSON.stringify({ type: 'ping' }));
    });

    socket.on('message', (data) => {
      check(JSON.parse(data), {
        'pong received': (d) => d.type === 'pong',
      });
      socket.close();
    });

    socket.setTimeout(() => socket.close(), 5000);
  });

  check(res, { 'connected': (r) => r && r.status === 101 });
}
```

# 整合 Grafana + InfluxDB

```yaml
# docker-compose.yml
version: "3"
services:
  influxdb:
    image: influxdb:1.8
    ports:
      - "8086:8086"
    environment:
      INFLUXDB_DB: k6

  grafana:
    image: grafana/grafana:latest
    ports:
      - "3000:3000"
    environment:
      GF_AUTH_ANONYMOUS_ENABLED: "true"
      GF_AUTH_ANONYMOUS_ORG_ROLE: Admin
    volumes:
      - grafana-data:/var/lib/grafana

volumes:
  grafana-data:
```

```bash
# 啟動
docker compose up -d

# 執行測試並輸出到 InfluxDB
k6 run --out influxdb=http://localhost:8086/k6 script.js
```

```
Grafana 設定：
1. 新增 Data Source → InfluxDB
   - URL: http://influxdb:8086
   - Database: k6
2. 匯入 Dashboard ID: 2587（k6 官方 Grafana Dashboard）
```
