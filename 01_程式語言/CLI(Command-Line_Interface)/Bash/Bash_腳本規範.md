# 基本結構

- Shebang 一律 `#!/usr/bin/env bash`；開頭必加 `set -euo pipefail`。
- 檔頭註解依序：用途（一行）、用法、旗標說明、前置需求。
- 執行順序固定：檔頭 → 設定區 → 參數解析 → 共用函式 → preflight → 主流程。
- 變數引用一律加大括號與引號：`"${VAR}"`。

# 設定區（可變參數集中最上方）

- 所有「可能因環境而改」的變數集中最上方，用醒目方框標示：

```bash
# ╔══════════════════════════════════════════════════════════════╗
# ║           ★ 需修改的設定（請依實際環境調整）★               ║
# ╠══════════════════════════════════════════════════════════════╣
  DEST_HOST=1.2.3.4
# ╚══════════════════════════════════════════════════════════════╝
# ↑↑↑ 以上為需修改處；以下一般不需更動 ↑↑↑
```

- 佔位值用可辨識字串（`YOUR_PASSWORD`），preflight 檢查到未改須終止。
- 密碼、token 走環境變數（`$ENV_VAR`），不提交明文進版控。

# preflight（執行前檢查）

主流程前依序檢查，任一不過即以**中文**提示並 `exit 1`：

1. 必要工具：`command -v` 檢查，缺少時印出安裝指令。
2. 佔位值未改：終止並提醒修改設定。
3. 遠端連線：`ssh -o ConnectTimeout=10 ... true`、`mysqladmin ping` 等先測通。
4. 選用工具（如 `pv`）缺少不終止，警告後降級執行。

```bash
log(){ echo "[$(date +%H:%M:%S)] $*"; }
die(){ log "✖ $*"; exit 1; }
need(){ command -v "$1" >/dev/null 2>&1 || die "缺少必要工具「$1」，請先安裝：$2"; }
```

# 訊息與進度

- 使用者訊息繁體中文；指令、變數、路徑保留英文。
- 各階段 `log` 印時間戳；結尾印 `=== DONE_XXX ===`。
- 長時間動作必有進度：打包用 `pv` 或 `tar --checkpoint`；傳輸用 `rsync --info=progress2`。

# 傳輸與重試

- 遠端傳輸一律 `rsync --partial`（禁裸 `scp`：無進度、無續傳）。
- 失敗自動重試，`RETRIES` / `RETRY_WAIT` 放設定區，包成 `send()` 函式。
- 密碼登入用 `sshpass -e`（密碼走 `SSHPASS` 環境變數）；可用金鑰時優先金鑰。
- 腳本不內建 tmux；需要斷線不中斷時由使用者以 tmux 包整支腳本。

# 慣用旗標

| 旗標 | 語意 |
|---|---|
| `--resend` | 跳過產生階段，直接重傳既有產物；找不到即終止 |
| `--cleanup` | 完成後遠端 `shasum -c` 驗證通過才刪本機暫存 |

- 解析用 `for a in "$@"; do case "$a" in ...`，位置參數與旗標順序不拘。

# 產物與驗證

- 暫存目錄統一 `/tmp/<用途>_<TAG>/`；`TAG` 預設 `date +%Y%m%d_%H%M%S`，跨主機共用同一 TAG。
- 傳輸前產 `shasum -a 256` 清單隨檔傳送，接收端 `shasum -c` 驗證。
- 不無條件刪產物；驗證通過（`--cleanup`）才可刪。

# 收尾要求

- 改完必跑 `bash -n`；可用時加跑 `shellcheck`。
- 每個腳本目錄配 `README.md`：分工表、執行順序、旗標、前置設定、驗證原理。
- `rm -rf` 前必有存在性檢查，禁止對未驗證變數路徑刪除。
