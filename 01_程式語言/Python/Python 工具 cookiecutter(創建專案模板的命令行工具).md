# 用法

## Cookiecutter 模板

```
my_project_template/
│
├── {{ cookiecutter.project_slug }}/
│   ├── __init__.py
│   ├── main.py
│   └── {{ cookiecutter.package_name }}/
│       └── __init__.py
├── tests/
│   ├── __init__.py
│   └── test_main.py
├── README.md
└── cookiecutter.json
```

```
{{ cookiecutter.project_slug }}：這是專案目錄，其名稱將由 Cookiecutter 提示輸入。
__init__.py：用於將目錄視為 Python 包的標準初始化文件。
main.py：專案的主要 Python 程序文件。
{{ cookiecutter.package_name }}：這是 Python 包目錄，其名稱也將由 Cookiecutter 提示輸入。
__init__.py：用於將目錄視為 Python 包的標準初始化文件。
tests/：測試文件目錄。
__init__.py：用於將目錄視為 Python 包的標準初始化文件。
test_main.py：主要 Python 程序的測試文件。
README.md：專案的說明文件。
cookiecutter.json：包含模板參數的 JSON 文件。這將提示用戶輸入專案的一些信息，例如專案名稱、Python 包名稱等。
```

cookiecutter.json 範例

```json
{
  "project_name": "MyProject",
  "author_name": "John Doe",
  "email": "john@example.com",
  "description": "A simple project created with Cookiecutter",
  "version": "0.1.0",
  "license": "MIT",
  "use_git": true,
  "dependencies": [
    "requests",
    "numpy"
  ]
}
```

cookiecutter.yaml 範例

```yaml
project_name: MyProject
author_name: John Doe
email: john@example.com
description: A simple project created with Cookiecutter
version: 0.1.0
license: MIT
use_git: true
dependencies:
  - requests
  - numpy
```

```
project_name：專案名稱。
author_name：作者名稱。
email：作者電子郵件地址。
description：專案描述。
version：專案版本。
license：專案授權。
use_git：是否使用 Git 作為版本控制系統。
dependencies：專案所需的依賴庫清單。
```

## 創建基本的 Python 專案結構

```bash
cookiecutter gh:cookiecutter/cookiecutter-pypackage
```

將提示輸入一些項目資訊，例如專案名稱、作者名稱等。

完成後，Cookiecutter 將生成一個新的專案目錄結構，其中包含了基本的 Python 專案文件和配置。

可以根據需要進一步修改和定製這個專案。

```
setup.py：用於定義專案的元資訊和安裝配置。
README.md：專案的說明文件。
LICENSE：專案的授權文件。
requirements.txt：列出了專案所需的所有依賴。
測試配置文件，例如 pytest.ini 或 tox.ini。
項目結構，包含了主要的 Python 代碼目錄結構。
```
