# 文件 J｜上下文經濟與 Token 預算（CONTEXT_ECONOMY）

> 版本：1.0（2026-07-07）
> 定位：A~I 治理「行為與安全」（派工、判斷、事故、權限、變更）；本文件治理**上下文與 token 成本本身**。四個已知漏點都指向這裡。與文件 B 的 S/W/R 分層直接掛鉤。
> 執行方式：任務開始前先依第 2 章估預算；執行中超標即降級或拆單；每次讀檔前先過第 3 章紀律。

| # | 漏點 | 根因 | 對治章節 |
|---|------|------|----------|

3. **會腐化的事實一律外部化**——不靠 session 記憶。

## 第 2 章：分層 Token 預算（掛 B 的 S/W/R）

預算為**單次任務**的輸入上下文軟上限，非硬限；觸頂即觸發右欄動作。

| 角色 | 模型層 | 上下文軟上限 | 觸頂動作 |
|------|--------|--------------|----------|
| S（摘要/巡檢） | Haiku | 小（僅任務單 + 目標片段） | 拆更小的子任務 |
| W（實作/修改） | Sonnet | 中（任務單 + 相關檔片段 + G 摘要） | ⬆ 升 R 評估是否需要拆單 |
| R（架構/決策） | Opus | 大（可含多檔與決策樹） | 先產「範圍收斂摘要」再繼續 |

判斷序（每次交辦前跑一次）：
```
這件事需要讀幾個檔？ → 只 1~2 檔片段可解 → 派 S/W
                     → 需跨多檔理解 → 先派 S 做摘要，再把摘要（非原文）交 W/R
估算輸入是否超該角色軟上限？ → 是 → 拆單 or 降精度（讀片段不讀全文）
```

## 第 3 章：檔案讀取紀律（對治漏點 1）

- **禁止**無定位就 `cat`／view 整個檔或整個目錄。
- 先定位、再讀片段：
  ```bash
  grep -rn "關鍵字" path/ --include="*.ext"   # 先找行號
  sed -n '120,180p' path/file                  # 只讀命中區段
  ```
- 需要結構總覽用 `find path -maxdepth 2 -type f` 或 `tree -L 2`，**不要**逐檔開。
- 讀進 context 的原文只保留「這一步用得到」的段落；用完的長貼文在下一輪摘要掉。

## 第 4 章：Agent Loop 歷史修剪（對治漏點 2）

- 每輪只保留：**固定系統摘要（1 份）＋ 最近 N 輪對話（建議 N≤6）**，其餘丟棄。
- 固定系統摘要內容 = 角色（S/W/R）＋當前任務目標＋不變的環境指標（引用 G，不展開）。
- 中途結論若之後還要用 → 寫進固定摘要或落地檔案，**不要**靠「歷史還在」。
- 迴圈超過設定輪數仍未收斂 → 依文件 C 清單 4 停止重試，改開新 session（見文件 K）。

---

## 第 5 章：`/compact` 門檻與壓縮後驗證（對治漏點 3）

觸發門檻：
- 主動觸發時機：context 逼近上限、或單一任務已跨 3 個以上子步驟。
- **禁止**在正式環境變更（文件 H 流程）進行到一半時 compact——先完成或先落地回滾步驟再壓。

壓縮後驗證清單（compact 完成後，AI 必須自檢仍在場）：
- [ ] 當前角色與任務目標
- [ ] 進行中變更的**回滾指令**（若有，來自 H）
- [ ] 本次已鎖定「只准讀」的檔案清單（來自 D）
- [ ] 尚未回報的中途結論
任一項不在場 → 從落地檔案（G / CLAUDE.md / 任務單）重讀補回，不得憑印象續作。

## 第 6 章：外部化優先原則（對治漏點 4）

「會腐化的事實」不得依賴 session，一律落地。落地位置對照：

| 事實類型 | 落地位置 | 依據 |
|----------|----------|------|
| 環境現狀（主機、版本、port、金鑰盤點） | 文件 G | G 為單一真相 |
| 專案級 AI 行為約束 | `CLAUDE.md`（見文件 L） | 隨 repo 版控 |
| 安裝／設定步驟 | skill 檔（見文件 M） | 步驟留 skill，現狀搬 G |
| 易變外部狀態（chat_id、host key、GPG、認證方式） | 文件 G 的「易腐清單」欄 | 明確標記需定期核對 |

原則：**步驟留 skill，現狀進 G，行為約束進 CLAUDE.md，會過期的東西標記到期核對**。session 內只引用指標，不重述內容。
