# 筆記規範

> 本 repo 建立與維護筆記的格式規範。範本檔在本目錄（`00_sample/`）。
> 行為原則（先查索引、單一真相等）定義在根目錄 `CLAUDE.md`。

## 範本對照表

| 筆記類型 | 使用範本 |
|---|---|
| 工具 / 服務（含安裝步驟） | `00_common_sample.md` |
| 概念 / 方法論（無安裝步驟） | `00_note_sample.md` |
| Python 語法筆記 | `python_sample.md` |
| JavaScript 語法筆記 | `javascript_sample.md` |
| MySQL 語法筆記 | `mysql_sample.md` |
| MongoDB 語法筆記 | `mongodb_sample.md` |
| Linux 指令筆記 | `linux_sample.md` |
| Go 語法筆記 | `golang_sample.md` |
| GitHub 相關筆記 | `github_sample.md` |
| 語言內建語法（模組 / 函式庫） | 對應 `{語言}_build-in_sample.md`（Python / Java / JavaScript / PHP / C++） |
| 其他語言/工具 | 本目錄同名 `*_sample.md`；沒有對應範本時用 `00_common_sample.md` |

- 範本開頭有多個 H1 標題（如 `模組` / `工具`）時，複製後依實際類型**擇一保留**，其餘 H1 與目錄中對應錨點一併刪除。

## 命名規則

筆記檔名後方須加上括號說明該工具 / 服務的用途或類型，一到三個詞即可：

```
格式：{名稱}({用途說明}).md

範例：
  Claude(AI).md
  Vite(前端開發工具).md
  Synology(NAS).md
  k3s(輕量K8s).md
  ProxySQL(MySQL代理).md
```

- 括號內使用繁體中文或通用縮寫（AI、NAS、CLI 等）
- **目錄名稱**同樣需要括號描述（如 `Claude(AI)/`、`爬蟲相關(網路爬蟲)/`）
- 純語法筆記（如 `Git_筆記.md`、`MySQL_筆記.md`）、`_總覽.md`、`_範例.md` 不強制加括號

## 結構要求

- 開頭用 ```` ```...``` ```` 區塊寫一行摘要
- 必須有 `## 目錄` 與 `## 參考資料` section
- 章節標題使用繁體中文，程式碼識別字保留英文
- 工具類筆記包含：安裝（docker-compose / Debian / RedHat / Homebrew / Windows）、配置文檔、指令、例外狀況（錯誤排除；工具/服務類範本已含此章節，沒內容可先留空）
- 語法類筆記（模組 / 套件）章節順序：安裝 → 指令 → 用法；安裝指令（pip / npm / composer…）放 `# 安裝`，不放 `# 指令`

## 跨分類筆記（單一真相 + 連結）

筆記同時符合多種分類（如 Docker 既是 DevOps 工具也與伺服器部署相關）時：

- 筆記本體只放**一個最符合的目錄**，不複製、不建副本。
- 其他相關筆記在 `## 參考資料` 底下建立**相對路徑連結**指向它：

```markdown
## 參考資料

[Docker 筆記](../../00_其他(工具_應用程式_網站)/DevOps(Development和Operations的組合詞)/Docker(開放平臺軟體)/Docker_筆記.md)
```

- 移動或改名被連結的筆記時，需 `grep -rn` 舊路徑，同步修正所有引用連結與 `00_索引.md`。

## 更新既有筆記（內容歸位）

補充內容前先看該筆記的章節結構（`grep -n "^#" 檔案`），把內容放進**語意相符的標題底下**，不要就近插入或往檔尾塞：

| 內容類型 | 應放章節 |
|---|---|
| 安裝指令（apt / yum / brew / docker-compose…） | `# 安裝`（或 `# 安裝部分`）下對應平台小節 |
| 設定檔內容、參數說明 | `# 配置文檔` |
| 操作指令、用法範例 | `# 指令` 或對應主題章節 |
| 錯誤訊息、異常排除、troubleshooting | `# 例外狀況` |

- 語意相符的章節不存在 → 依範本結構補該章節，再放內容。
- 新增章節後同步更新該筆記的 `## 目錄`。

反例（實際發生過）：更新 iptables 筆記時，將安裝指令 `apt install iptables-persistent` 加在 `# 配置文檔` 的 `## Debian / Ubuntu（netfilter-persistent）` 底下——正確位置是 `# 安裝部分 → ## 安裝步驟 Debian / Ubuntu`。

## 總覽筆記規則

當一個目錄下收錄了**同類型的多個工具**時，建立 `XXX_總覽.md` 作為導覽入口。

### 命名

`{目錄名稱}_總覽.md`，放在該目錄的根層。

### 必要章節

| 章節 | 說明 |
|---|---|
| 開頭摘要區塊 | 說明此目錄收錄哪類工具 |
| `## 目錄` | 含錨點連結的 TOC |
| `## 參考資料` | 連結至子目錄各筆記 |
| **工具清單** | 表格列出所有工具，含類型、主要用途、筆記連結 |
| **類型 / 特性比較表** | 以表格對比各工具的核心差異（架構、效能、適用情境等） |
| **各工具說明** | 每個工具一節，說明定位、適用 / 不適用情境，有子工具時列出生態連結表 |
| **需求選擇建議** | 以情境為列，對應推薦工具，說明理由 |

### 選填章節（視工具性質加入）

| 章節 | 適用時機 |
|---|---|
| **HA / 叢集方案總覽** | 工具有多種 HA 或叢集部署模式時（資料庫、快取） |
| **常見部署組合** | 工具通常需要搭配其他工具使用時（Web Server、Application Server） |
| **角色區分** | 目錄內工具分屬不同層次（如 Web Server vs App Server）時 |
